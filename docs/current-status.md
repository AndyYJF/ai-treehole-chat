# 当前状态

## 已实现

- 单用户 Web 树洞聊天。
- Fable 5 辅助重构后的暖纸/松林风格前端，覆盖聊天、登录、设置、记忆和用量面板。
- DeepSeek V4 主模型路由：自动、轻声、均衡、深谈。
- DeepSeek 真实流式响应，前端逐段更新助手消息。
- DeepSeek 调用重试、用量记录和缓存命中统计。
- 设置面板内可查看 token、缓存命中率、延迟和请求数。
- LangGraph 风格服务端聊天流程。
- LangMem 风格长期记忆抽取、存储、检索和 prompt 注入。
- DeepSeek JSON 记忆抽取，失败时规则兜底。
- SiliconFlow rerank 记忆检索，失败时稳定排序兜底。
- 服务端 Postgres 持久化：
  - `chat_threads`
  - `chat_messages`
  - `memories`
  - `user_memory_settings`
  - `model_usage_events`
- 多设备访问同一实例时同步多条聊天时间线和同一份长期记忆。
- 用户可在时间线面板中新建、切换历史时间线，并清空当前时间线上下文。
- 用户可查看、编辑、删除、清空、暂停记忆；记忆写入后默认生效。
- 用户可导出聊天、记忆和记忆设置。
- 用户导出的数据包含模型用量摘要和最近调用记录。
- 用户可清空模型用量记录。
- 用户可一键清空聊天、长期记忆和模型用量记录。
- 用户可退出当前访问会话。
- Docker Compose 部署，Postgres 仅 Docker 内网可访问。
- 单用户访问口令门禁，使用 httpOnly cookie 保存会话。
- frontend-design 兼容性调研文档。

## 当前设计取舍

- 这是单用户私有实例，不做账号体系。
- 不做端到端加密；私密性依赖自托管、数据库不暴露公网、服务端环境变量保护。
- 聊天记录按时间线隔离；清空当前上下文只删除当前时间线消息，保留长期记忆和其他时间线。
- 前端只保留档位、温度、记忆开关等偏好，不再把聊天记录作为权威存储。
- 流式回复结束后才写入完整助手消息；测试或失败的半截回复不会落库。
